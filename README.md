# SAGA — Stateful Context Engine for Narrative RP

AI 캐릭터와의 장기 대화에서 **프롬프트 캐시를 깨지 않고** 맥락에 맞는 기억을 회상하는 리버스 프록시.
RisuAI · SillyTavern 등 기존 롤플레이 프론트엔드 수정 없이 삽입할 수 있다.

---

## 핵심 지표

| 지표 | 결과 | 비고 |
| --- | --- | --- |
| LOCOMO (ACL 2024) | **+54%** | judge 2.02 → 3.12, multi-hop 4.71/5 |
| LongMemEval (ICLR 2025) | **+42.3%p** | 21.2% → 63.5% |
| 프롬프트 캐시 적중률 | **84.2%** | steady-state 86.8% |
| 순비용 절감 | **67.1%** | cache_read 단가 + 프롬프트 단축 합산 |

## 무엇을 푸는가

AI 캐릭터 챗은 대화가 길어질수록 세 가지 문제가 반복된다.

- **성격 희석** — 30턴 이후 캐릭터가 평균적인 LLM 어시스턴트로 수렴
- **관계 리셋** — "어제 싸운 일" 같은 관계 상태가 휘발
- **서사 유실** — 복선·장소 이동·등장 NPC가 attention 범위 밖으로 나가면 사실상 소실

기존 솔루션(HypaMemory, Letta, mem0)은 매 턴 기억을 재조립해 **프롬프트 캐시를 무력화**하거나 턴당 3~5회 LLM 호출로 **레이턴시·비용이 폭증**한다.
SAGA는 캐시 보존과 회상 정확도를 동시에 확보하는 구조로 이 문제를 푼다.

## 아키텍처

```
═══ CACHE PREFIX (불변) ════════════════════════════════
  [System Prompt] → [.md prefix: 세계관/NPC/상태] → [과거 summary chunks]
═══════════════════════════════════════════════════════

  ▼ 동적 영역 (매 턴 변경)
  [검색 결과 prepend] + [최근 user 메시지]

  ▼ 메인 LLM 1회 호출 (유저 응답 경로)
  [응답 반환 → 유저]

  ▼ 백그라운드 (응답 후)
  ├─ Sub-B 서사 요약 (경량 모델 1회, 4필드 구조)
  ├─ SQLite 업데이트 (관계·이벤트·Lorebook)
  ├─ ChromaDB 색인 (에피소드 임베딩)
  ├─ MessageCompressor (임계치 초과 시 chunk 압축)
  └─ Curator (모순 검증·lore 생성)
```

유저 체감 레이턴시는 메인 LLM 1회. 기억 갱신 작업은 응답 반환 이후 백그라운드에서 처리되어 체감 경로 밖에 있다.

## 저장소 3계층

| 저장소 | 용도 | 쿼리 특성 |
| --- | --- | --- |
| **SQLite** | 관계 그래프 · 이벤트 체인 · Lorebook · 세계관 상태 | 정확한 조건 조회 |
| **ChromaDB** | 에피소드 단위 대화 기억 (임베딩) | 의미 기반 유사도 검색 |
| **.md prefix** | 세계관 · 현재 상태의 Markdown 직렬화 | Anthropic 3-Breakpoint 캐싱 |

단일 저장소로는 "NPC 관계 상태" 같은 구조적 조회와 "숲에서 있었던 일" 같은 의미 기반 회상을 동시에 할 수 없다.
용도가 다르면 저장소도 다르다.

## 핵심 설계 한 줄 요약

- **캐시 불변성** — 검색 결과 같은 동적 컨텍스트는 마지막 user 메시지 prepend로만 주입. system prefix는 절대 안 건드림
- **단일 동기 호출** — 유저 응답 경로에는 메인 LLM 1회만. 요약·색인·검증은 전부 비동기
- **요약 1회 → 다용도 재활용** — Sub-B의 4필드 서사 요약 1회를 압축·색인·관계 업데이트·검증 4곳에서 공유
- **Immutable summary chunk** — 압축된 chunk는 동결. 매번 재요약하지 않아 prefix 해시가 안정
